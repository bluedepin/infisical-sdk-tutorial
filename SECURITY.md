# Security Policy

## Reporting a Vulnerability

If you discover a security issue in any code, configuration, or instruction in this tutorial, please report it responsibly:

- For non-sensitive issues (e.g. an outdated dependency, a misleading instruction that could lead a reader to a less-secure setup), open a public GitHub issue with the `security` label.
- For sensitive issues that should not be disclosed publicly until fixed, open a draft security advisory under the repository's "Security" tab, or contact the repository owner privately via their GitHub profile.

Please do not exploit any issue you find, and do not test against systems you do not own.

## Supported Versions

This is a tutorial repository. The instructions are tested against the pinned tool versions in `prerequisites.json` and the pinned package versions in any `requirements.txt` shown in the tutorial. Older versions are not supported; readers should use the pinned versions or check for updates.
